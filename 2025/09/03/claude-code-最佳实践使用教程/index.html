🧩 概览

本文整理自 Anthropic 官方文章“Claude Code: Best practices for agentic coding”，并结合实际终端操作给出一套可复制的使用教程，帮助你把 Claude Code 融入日常开发工作流。

参考链接（建议收藏）：Claude Code: Best practices for agentic coding

🔧 环境与基础配置

  • 安装与启动

    • 推荐使用最新版 Claude Code（命令行工具）。
    • 将项目根目录作为你的工作目录，便于工具自动收集上下文。

  • 创建与维护 CLAUDE.md

    • 作用：在会话开始时自动被读取，充当“项目速记 + 使用说明”。
    • 放置位置：
      • 仓库根目录（最常见）
      • 子目录（单体仓库的子项目）
      • 父目录（在多层级项目中复用）
      • 用户目录 ~/.claude/CLAUDE.md（全局生效）
    • 建议内容：常用命令、代码规范、测试与部署方式、注意事项等。

  • 精调 CLAUDE.md

    • 把它当“提示词”维护，短平快且可迭代。
    • 经常回顾有效性，结合团队实践不断修改。

🛡️ 工具授权与安全策略

  • Claude Code 默认保守：涉及改动系统或代码的行为会请求授权。
  • 你可以通过以下方式放行常用、安全的工具：
    • 在授权弹窗中选择“Always allow”。
    • 会话内使用 /permissions 管理允许列表。
    • 编辑 .claude/settings.json 或 ~/.claude.json。
    • 启动时使用 --allowedTools 临时放行。

常见建议：允许 Edit（文件编辑）、Bash(git commit:*)（提交易回滚），其余按需放开。

🧰 让 Claude 拥有更多“本地超能力”

  • 继承你的 Shell 环境：可直接使用系统工具与脚本。

  • 使用 MCP（Model Context Protocol）：

    • 作为客户端连接多个 MCP 服务器（如浏览器自动化、监控平台等）。
    • 可放在项目配置、全局配置或 .mcp.json 中共享给团队。

  • 自定义斜杠命令 /：

    • 在 .claude/commands 中放置 Markdown 模板，作为可复用工作流。
    • 支持 $ARGUMENTS 参数占位，便于快速复用。

🧠 高效工作流范式

  • 清晰拆分任务：复杂任务用清单（Markdown/GitHub Issue）驱动。

  • 多 Claude 协作：

    • 一个写代码，另一个评审或写测试；再开一个根据评审修改。
    • 多分支/多工作树（git worktree）并行推进互不干扰的任务。

  • Headless（无头）模式自动化：

    • claude -p "<你的提示>" --output-format stream-json 适合集成到 CI/脚本。
    • 两种模式：
      • 扇出（fan-out）：大规模迁移/批处理分析。
      • 流水线（pipeline）：把 Claude 嵌入既有数据/处理链路。

🧪 给 Claude 提供正确的数据

  • 常用方式：直接粘贴、管道输入、让它自行拉取（bash/MCP/HTTP）、读取本地文件。
  • 要点：一次只专注一件事，必要时 /clear 重置上下文以避免干扰。

🛠️ 实用命令速记（可放进 CLAUDE.md）

🚀 新手到进阶的实操路线

  1. 初始化与环境：

    • 在项目根新建简洁的 CLAUDE.md，记录常用命令与规范。
    • 配置允许列表，放行文件编辑与 git commit。

  2. 日常开发：

    • 让 Claude 打开/阅读关键文件，提出设计或重构建议。
    • 以“编辑-测试-修复”的小循环推进，降低上下文噪声。

  3. 自动化提效：

    • 为重复操作沉淀 / 命令模板（如“修复某类 Lint 错误”、“生成发布日志”）。
    • 在 CI 中用 headless 模式做问题分拣、文本校对、变更摘要。

  4. 团队协作：

    • 把 CLAUDE.md 与 .claude/commands 一并纳入版本控制，团队共享。
    • 用 git worktree/多工作目录让多个 Claude 并行处理不同子任务。

📎 参考与延伸阅读

  • 官方最佳实践原文：
    • Claude Code: Best practices for agentic coding

如果你正把 Claude Code 集成到本博客的写作/发布流程，我也建议把常用命令与注意事项写进仓库根的 CLAUDE.md，这样每次开启对话都能自动获得最佳上下文，效率会更高。
